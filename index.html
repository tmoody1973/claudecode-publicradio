Claude Code for Public Radio

A role-by-role guide for stations that do more with less. Eight desks, one dial — tune to yours.

Why this guide exists

Built for small teams wearing many hats

Public radio runs on small teams wearing many hats. Claude Code — Anthropic's agentic AI tool — was built for software developers, but it turns out to be a force multiplier for anyone whose job involves files, data, documents, and repeatable workflows. That's everyone at your station.

Claude Code works directly with the files on your computer. Unlike a chatbot that only talks, it can read your spreadsheets, draft and organize documents, analyze data, automate repetitive tasks, and write software. You describe the outcome in plain language; it plans the approach, does the work, and shows you the results. It's available in the Claude desktop app (easiest start), in your browser at claude.ai, in editors like VS Code, and in the terminal.

Small staff, big surface area

No developers, no analysts, no spare hands? A membership manager can clean a donor file, a music director can analyze spin data, a PD can prototype a tool — without hiring for it.

Your data is already files

Nielsen PDFs, Triton reports, CRM exports, playlists, grant narratives, board decks. Claude Code works where that data already lives — no pasting into a chat box.

Repeatable work is automatable work

Weekly metrics decks, acknowledgments, program logs, show prep. Anything you do the same way every week can become a saved workflow you run with one command.

Mission alignment matters

AI should extend your staff, not replace your voice. Use it for operational work — and keep commissioned creative, on-air talent, and editorial judgment human and local.

Setup

Getting started — no terminal required

Before you start: you need a paid Claude plan (Pro, Max, Team, or Enterprise) or an Anthropic Console account with API credits. The free plan doesn't include Claude Code. Official docs: code.claude.com/docs

Beginner — desktop app or browser

Download the Claude desktop app or open Claude Code at claude.ai. No terminal, no install commands. The right starting point for most station staff.

Comfortable — VS Code

Install the Claude Code extension from the Extensions marketplace for inline diffs and conversation history next to your work.

Power user — the terminal

Install the native version with the one-line command from the official docs, or via npm with Node.js 18+: npm install -g @anthropic-ai/claude-code. Then type claude in any project folder and log in on first use.

The beginner path, step by step — desktop app or browser

The right starting point for most station staff. No terminal, no install commands, nothing to configure.

  1. Confirm your account. You need a paid Claude plan — Pro, Max, Team, or Enterprise. On a station Team plan? Ask whoever manages it to add you.
  2. Get the app (or skip to the browser). Download the Claude desktop app for Mac or Windows from claude.ai/download and sign in — or run Claude Code in your browser at claude.ai with nothing installed.
  3. Start a session in a real folder. Point Claude Code at a folder of real files — start with copies, like this week's reports. It asks permission, scoped to the folder you choose.
  4. Describe the outcome, not the steps. "Read these three PDFs and build a one-page summary comparing them." It plans its approach and shows you before acting.
  5. Approve, review, iterate. It asks before doing anything consequential. Check the result like you'd check a new colleague's first draft.
  6. Make it repeatable. When something works, ask Claude Code to write down the steps it took so you can run the same workflow next week — the seed of your first skill.
The CLI path — for the technically comfortable
  • —Recommended: run the one-line native installer from the official setup page at code.claude.com/docs — no dependencies, auto-updates.
  • —Alternative: with Node.js 18+, npm install -g @anthropic-ai/claude-code. Note: npm installs don't auto-update.
  • —Windows native installs also need Git for Windows (git-scm.com). WSL setups don't.
  • —Open a terminal, cd into any project folder, and type claude. You'll log in via your browser on first use; /login switches accounts later.
  • —Type /init in your first session and Claude drafts a CLAUDE.md for the project.

Working in VS Code? Install the Claude Code extension from the Extensions marketplace (Cmd/Ctrl+Shift+X, search "Claude Code"), then open it from the Command Palette for inline diffs beside your files.

A note on freshness: install commands are the fastest-staling content in any guide. If anything here disagrees with code.claude.com/docs, the docs win.

Claude Code or Claude Cowork? Same engine, two front doors

Claude Cowork brings Claude Code's agentic architecture to everyday knowledge work — no terminal, no code. In the Claude desktop app, select Cowork in the message box and hand it a task; it plans, works through subtasks, and delivers finished files for review. Sessions run remotely, so work continues when your laptop is closed — and it's rolling out on web and mobile too. Same paid plans as Claude Code.

Start in Cowork for…

  • —Reports and briefs from messy inputs — notes, transcripts, voice memos
  • —Slide decks and spreadsheets with working formulas, not just CSVs
  • —Donor-file analysis, reconciliations, "pull these updates into one document"
  • —Scheduled tasks: the Monday metrics brief that builds itself

Start in Claude Code for…

  • —Building something: scripts, tools, websites, data pipelines, MCP integrations
  • —Deep work inside a project folder with CLAUDE.md, skills, and version control
  • —Engineering and automation workflows where the terminal is home

Rule of thumb: if the deliverable is a document, deck, or spreadsheet, start in Cowork. If it's a tool or system, start in Claude Code. Anthropic's own data: more than 90% of Cowork sessions aren't software development — the top categories are business operations and content creation. That's a public radio week in a nutshell. Most starter use cases in the role guide run beautifully in Cowork; the power-user paths live in Claude Code.

Five concepts that unlock everything else

CLAUDE.md

A plain-text file read at the start of every session. Put your style guide, brand voice, report formats, and "never do this" rules in it once — every session starts briefed. Type /init and Claude drafts it for you.

Plan Mode

Ask Claude Code to plan first: it lays out intended steps and waits for approval before touching anything. Ideal when you're new or stakes are high.

Skills

A saved, shareable recipe — /weekly-metrics, /show-prep — anyone on staff can run. One person's workflow becomes the whole station's capability.

MCP connections

The Model Context Protocol lets Claude Code read Google Drive, pull from Slack, update project boards, and connect to broadcast and fundraising systems. Your data stays put; Claude comes to it.

Auto memory

Claude Code saves what it learns as it works — commands, quirks of your files, decisions you made — so future sessions start smarter.

Station policy

Responsible use — the public-media way

Public radio's asset is trust. Five commitments to adapt into your station's written policy:

  • Keep creative and editorial human. Draft with AI, decide with people. On-air voice, commissioned creative, and editorial judgment stay human — and where possible, local.
  • Verify before it airs. Claude Code accelerates research; it doesn't replace verification. Ask for page numbers and sources so checking is easy.
  • Protect donor and listener data. Treat exports with the same care as the systems they came from. Work on copies. Follow your organization's data policy.
  • Review before it acts. Use Plan Mode when stakes are high. Keep backups. Read what it proposes, especially early on.
  • Start small, share what works. One task, one folder, one week. When a workflow works, turn it into a skill and hand it to a colleague.

The one-week challenge. Pick one task you do every week. Put its files in a folder. Spend 30 minutes with Claude Code on it. If it saves you an hour, teach it to a colleague — that's how a station's capability compounds.
